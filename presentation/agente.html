03 · Agente

L'agente in tre componenti

Un singolo LlmAgent del Google ADK eseguito su Vertex AI: modello, system prompt e tool, orchestrati da un loop ReAct.

modello LLM

Modello Gemini

Modello generativo di Google su Vertex AI, integrato tramite il Google ADK.

  • Default: gemini-2.5-pro
  • Alternativa: gemini-2.5-flash
  • Selezionabile dall'utente prima del run
istruzioni

System prompt

Vincola il modello al dominio del testing ETL.

  • 6 fasi: lettura → pre-check → piano → esecuzione → fix+PR → report
  • 7 regole anti-allucinazione su SQL
  • Aggiornabile senza riaddestrare il modello
tool

Otto funzioni Python

Registrate nell'ADK come FunctionTool: unico canale verso l'esterno.

  • Lettura del codice SQL su GitHub e dei dati su BigQuery (sola lettura)
  • Pubblicazione degli artefatti del run: piano, risultati, proposta di fix
  • Apertura della Pull Request e salvataggio del report finale
ragionamento

Il loop ReAct

ReAct (Reasoning + Acting): il modello alterna ragionamento e azione, raccoglie informazioni dai tool e aggiorna il contesto fino al report finale.

Le tre fasi di ogni iterazione

  • Reason — il modello sceglie l'azione: tool da invocare o fine.
  • Act — emette una function_call; l'ADK esegue la funzione Python e replica gli eventi al frontend via SSE.
  • Observe — la function_response rientra nel contesto: il modello la legge e decide il passo seguente.

Termina con save_final_report o al limite di 40 iterazioni; l'utente può cancellare in qualsiasi momento.

Esempio · "verifica i saldi correnti"

REASON "leggo il SQL dell'ETL"
ACT read_sql_code("saldi_correnti")
OBSERVE file SQL ricevuti
REASON "verifico le tabelle del join"
ACT execute_bigquery_query("… INFORMATION_SCHEMA.TABLES")
OBSERVE tabelle presenti
… e così via, fino al report o alla PR.
