Deep-dive · Frontend

Frontend Next.js

UI tematica scura in italiano. Due pagine, streaming live degli eventi dell'agente via Server-Sent Events.

Stack

  • Next.js 15 con App Router; React 19; TypeScript.
  • Tailwind CSS + shadcn/ui per i componenti; icone Lucide.
  • Static export: npm run build produce out/, copiato dal Dockerfile dentro il backend e servito come asset statici da FastAPI.
  • Niente CDN, niente CORS: stesso processo, stesso porto del backend.

Le due pagine

Route /

Storico richieste

  • Tabella con tutti i run (storico).
  • Stato (pending / running / completed / failed / cancelled), ETL coinvolto, timestamp relativo.
  • Polling ogni 4s per aggiornare l'elenco.
  • Sheet laterale per nuova richiesta (scorciatoia N).
Route /run?id=...

Timeline live del run

  • Eventi in tempo reale: ragionamento, tool call, tool response.
  • Render delle card-artefatto: piano test, risultati, fix proposal, PR aperta, report finale.
  • Pulsante Interrompi che cancella il task asyncio sul backend.
  • Reconnect-friendly: alla riconnessione il backend rigioca lo storico eventi dal documento Firestore.

Streaming · Server-Sent Events

La pagina /run apre un EventSource verso il backend e riceve un flusso continuo di eventi JSON, ognuno con un type: agent_text, tool_call, tool_response, artifact, status. 

const es = new EventSource(`/api/runs/${id}/stream`);
es.onmessage = (e) => {
  const ev = JSON.parse(e.data);
  // ev.type: agent_text | tool_call | tool_response | artifact | status
  appendToTimeline(ev);
};
