Deep-dive · Backend

Backend FastAPI

Espone le API REST e lo streaming SSE al frontend; per ogni richiesta lancia un task asyncio che fa girare l'agente ADK; persiste tutto su Firestore.

Stack

  • FastAPI + uvicorn · Pydantic v2 per i modelli.
  • Google ADK (google-adk) per l'agente; google-cloud-aiplatform per Vertex AI.
  • google-cloud-bigquery e google-cloud-firestore per i client GCP.
  • PyGithub per leggere il SQL e aprire le PR.
  • Python 3.11, deploy su Cloud Run.

Endpoint REST + SSE

POST   /api/runs                 → crea un run e lo lancia in background
GET    /api/runs                 → lista storico (limit 50)
GET    /api/runs/{id}            → dettaglio + eventi + artefatti
DELETE /api/runs/{id}            → elimina un run
DELETE /api/runs                 → pulisce tutto lo storico
POST   /api/runs/{id}/stop       → cancella il task asyncio
GET    /api/runs/{id}/stream     → SSE: replay + eventi live
GET    /api/etls                 → lista cartelle ETL nel repo
GET    /healthz                  → liveness probe

Run in background

Pattern

  • asyncio.create_task per ogni richiesta: il client riceve subito il run_id.
  • Un ContextVar propaga il run_id ai tool senza passarlo a mano.
  • Il task è registrato in un dict {run_id: task} per consentire la cancellazione cooperativa.
  • CancelledError gestita: se l'utente preme "Interrompi", il run chiude pulito con stato cancelled.

Persistenza · Firestore

  • Collection etl_agent_runs: un documento per run.
  • Eventi salvati con ArrayUnion sul campo events.
  • Artefatti strutturati salvati come campi nominati: artifacts.test_plan, artifacts.test_results, ecc.
  • Alla riconnessione SSE il backend rigioca dal documento e poi continua live.

Bus SSE in-memory

Per ogni run è mantenuto un set di code asyncio.Queue (una per client connesso). Quando un evento è prodotto:

  1. scritto su Firestore (append_event);
  2. messo in tutte le code dei subscriber (fan-out non bloccante);
  3. la connessione SSE invia un keepalive ogni 15s per evitare timeout proxy.
