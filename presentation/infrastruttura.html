02 · Infrastruttura

L'infrastruttura in tre blocchi

Tre componenti distinti, collegati tra loro: i dati su BigQuery, l'app agentica su Cloud Run, il codice SQL versionato su GitHub.

01

BigQuery · il data layer

Tre livelli di tabelle: sorgente lette dagli ETL, staging intermedie, finali prodotte come output. L'agente le interroga via service account dedicato, in sola lettura.

02

Cloud Run · l'app agentica

Frontend e backend deployati come container su Cloud Run. Il backend orchestra l'agente sfruttando Vertex AI per il modello Gemini e Google Agent Development Kit (ADK) per il loop ReAct.

03

GitHub · il codice ETL

Il repository ospita il codice SQL degli ETL, una cartella per pipeline. Per la demo abbiamo iniettato volutamente alcuni bug — formule errate, join sbagliati — così l'agente ha qualcosa di concreto da trovare.

04

Bus eventi · tracciabilità in tempo reale

Ogni passo dell'agente è persistito su Firestore e replicato al frontend via SSE. L'utente vede l'esecuzione passo-passo; gli eventi restano disponibili per audit anche dopo il run.
