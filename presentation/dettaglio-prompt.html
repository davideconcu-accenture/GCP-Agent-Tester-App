Deep-dive · System Prompt

Il prompt dell'agente

Un singolo LlmAgent guidato da un system prompt strutturato. Sei fasi obbligatorie, sette regole anti-allucinazione.

Le 6 fasi del workflow

FASE 1
Comprensione
list_available_etls · read_sql_code
FASE 2
Pre-check
una sola query per dataset su INFORMATION_SCHEMA
FASE 3
Pianificazione
3-10 test · emit_test_plan
FASE 4
Esecuzione
retry x3 · emit_test_results
FASE 5
Fix & PR
solo se FAIL/ERROR · create_github_pr
FASE 6
Report
save_final_report · sempre

Le 7 regole critiche

Sono i sassolini su cui un LLM scivola: scolpirli nel prompt evita problemi.

R1   No auto-confronto

Mai confrontare una colonna con la sua stessa formula del codice in test. Riferimenti indipendenti.

R2   Solo SELECT finale

Usa solo colonne del SELECT finale del CREATE TABLE, non quelle delle CTE intermedie.

R3   NULL check su LEFT JOIN

I campi che arrivano da LEFT JOIN possono essere NULL: vanno sempre filtrati.

R4   Niente invenzioni

Si valuta solo ciò che torna da execute_bigquery_query. No numeri inventati.

R5   Italiano

UI, report e PR in italiano. Inglese solo nel SQL.

R6   Mai cambiare dataset nel fix

Se una tabella manca dove il SQL la cerca, è un problema di ambiente, non un bug del codice.

R7   Stop in Fase 2 = stop totale

Se il pre-check rileva tabelle mancanti l'agente termina con il solo report finale: niente test, niente fix, niente PR.
